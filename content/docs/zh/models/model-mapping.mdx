## 概述

模型映射功能允许您：
- 使用 OpenAI 模型名称调用不同供应商
- 创建模型别名简化调用
- 将特定模型路由到首选供应商或账户

## 添加映射

1. 导航到 **模型管理** > **模型映射**
2. 点击 **添加映射**
3. 配置映射规则
4. 保存更改

## 映射配置

### 基本映射

| 字段 | 描述 |
|------|------|
| 请求模型 | API 请求中的模型名称 |
| 实际模型 | 发送给供应商的模型名称 |
| 首选供应商 | （可选）指定使用的供应商 |
| 首选账户 | （可选）指定使用的账户 |

### 示例

将 `gpt-4` 映射到 `DeepSeek-V3.2`：

| 请求模型 | 实际模型 | 供应商 |
|----------|----------|--------|
| `gpt-4` | `DeepSeek-V3.2` | DeepSeek |

```bash
# 使用 gpt-4 请求
curl http://localhost:8080/v1/chat/completions \
  -d '{"model": "gpt-4", "messages": [...]}'

# 实际使用 DeepSeek 的 DeepSeek-V3.2
```

## 通配符映射

使用 `*` 作为通配符匹配多个模型。

### 示例

| 模式 | 匹配 |
|------|------|
| `gpt-*` | gpt-4, gpt-3.5-turbo, gpt-4o |
| `gpt-4*` | gpt-4, gpt-4o, gpt-4-turbo |
| `*-turbo` | gpt-3.5-turbo, gpt-4-turbo |

<Callout type="info">
  通配符映射按顺序评估。更具体的模式应该放在前面。
</Callout>

## 优先级规则

当多个映射可能匹配时：

1. 精确匹配优先于通配符
2. 较早的规则优先于较晚的规则
3. 供应商特定规则优先于全局规则

### 优先级示例

| 顺序 | 请求模型 | 实际模型 | 优先级 |
|------|----------|----------|--------|
| 1 | `gpt-4` | `DeepSeek-V3.2` | 高（精确） |
| 2 | `gpt-*` | `GLM-5` | 低（通配符） |

## 供应商偏好

将特定模型路由到首选供应商：

| 请求模型 | 实际模型 | 供应商 |
|----------|----------|--------|
| `gpt-4` | `DeepSeek-V3.2` | DeepSeek |
| `gpt-4` | `GLM-5` | GLM |

<Callout type="info">
  如果首选供应商不可用，系统会尝试其他供应商。
</Callout>

## 账户偏好

路由到供应商内的特定账户：

| 请求模型 | 实际模型 | 供应商 | 账户 |
|----------|----------|--------|------|
| `gpt-4-premium` | `DeepSeek-V3.2` | DeepSeek | 高级账户 |

## 使用场景

### OpenAI 兼容性

将 OpenAI 模型名称映射到其他供应商：

| 请求 | 实际 |
|------|------|
| `gpt-4` | `DeepSeek-V3.2` |
| `gpt-3.5-turbo` | `GLM-5` |
| `gpt-4-vision` | `Qwen3-VL-235B-A22B` |

### 模型别名

创建别名以便于引用：

| 请求 | 实际 |
|------|------|
| `fast` | `DeepSeek-V3.2` |
| `smart` | `Kimi-K2.5` |
| `code` | `Qwen3-Coder` |

### 测试

将测试请求路由到特定账户：

| 请求 | 实际 | 账户 |
|------|------|------|
| `test-model` | `DeepSeek-V3.2` | 测试账户 |

## 管理映射

### 编辑映射

1. 在列表中找到映射
2. 点击 **编辑**
3. 修改设置
4. 保存更改

### 删除映射

1. 在列表中找到映射
2. 点击 **删除**
3. 确认删除
