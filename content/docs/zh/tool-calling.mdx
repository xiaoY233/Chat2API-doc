## 概述

Chat2API 为不支持原生工具调用的模型提供函数调用模拟功能。这允许您在任何供应商上使用 OpenAI 兼容的工具调用格式。

## 工作原理

系统通过以下方式工作：

1. **Prompt 注入**：将 OpenAI 格式的 `tools` 定义转换为系统提示词
2. **协议解析**：在流式响应中拦截协议文本
3. **格式转换**：将协议输出转换为 OpenAI 工具调用格式

## 注入模式

在 **高级设置** → **工具调用** 中配置注入模式：

| 模式 | 描述 |
|------|------|
| `auto` | 自动检测客户端类型，已知客户端跳过注入 |
| `smart` | 仅在请求看起来复杂时注入 |
| `always` | 始终注入工具定义 |
| `never` | 禁用注入，客户端已注入时使用 |

### 智能模式触发条件

智能模式在以下情况注入提示词：

- 消息长度超过阈值（默认 50 字符）
- 包含触发关键词：`search`, `find`, `get`, `call`, `use`, `tool`, `query`, `fetch`, `read`, `write`, `list`, `delete`, `update`, `create`
- 包含问号（`?` 或 `？`）
- 包含代码块（``` 或 `code`）
- 匹配行动模式：`help me ...`, `can you ...`, `please ...`, `i need to ...`, `i want to ...`

### 推荐模式

| 场景 | 推荐模式 |
|------|----------|
| 通用场景 | `auto` - 检测客户端并跳过已注入的 |
| 节省 token | `smart` - 保持功能的同时减少 token 消耗 |
| 一致行为 | `always` - 确保工具定义始终存在 |
| 客户端自行注入 | `never` - 禁用服务端注入 |

## 协议格式

系统支持多种协议格式：

### 括号格式（主要）

```
[function_calls]
[call:function_name]{"arg": "value"}[/call]
[/function_calls]
```

### XML 格式

```xml
<tool_use>
  <name>function_name</name>
  <arguments>{"arg": "value"}</arguments>
</tool_use>
```

### Anthropic 格式

```xml
<antml:function_calls>
  <antml:invoke name="function_name">
    <antml:parameters>{"arg": "value"}</antml:parameters>
  </antml:invoke>
</antml:function_calls>
```

## 使用示例

### 请求

```bash
curl http://localhost:8080/v1/chat/completions \
  -H "Content-Type: application/json" \
  -H "Authorization: Bearer your-api-key" \
  -d '{
    "model": "DeepSeek-V3.2",
    "messages": [
      {"role": "user", "content": "北京天气怎么样？"}
    ],
    "tools": [
      {
        "type": "function",
        "function": {
          "name": "get_weather",
          "description": "获取指定地点的当前天气",
          "parameters": {
            "type": "object",
            "properties": {
              "location": {
                "type": "string",
                "description": "城市名称"
              }
            },
            "required": ["location"]
          }
        }
      }
    ]
  }'
```

### 响应

```json
{
  "choices": [{
    "message": {
      "role": "assistant",
      "content": null,
      "tool_calls": [{
        "id": "call_abc123",
        "type": "function",
        "function": {
          "name": "get_weather",
          "arguments": "{\"location\": \"北京\"}"
        }
      }]
    }
  }]
}
```

## 配置

在 **高级设置** → **工具调用** 中配置：

| 选项 | 描述 | 默认值 |
|------|------|--------|
| 注入模式 | 工具定义注入策略 | `smart` |
| 智能阈值 | 智能模式的消息长度阈值 | 50 |
| 客户端检测 | 启用客户端类型检测 | 启用 |
| 提示词变体 | 提示词模板变体 | `default` |

## 最佳实践

| 实践 | 说明 |
|------|------|
| 清晰的描述 | 提供清晰的函数和参数描述，帮助模型理解工具用途 |
| 必需参数 | 明确标记必需参数，避免模型遗漏关键信息 |
| 简单的 Schema | 保持参数 Schema 简单，复杂的 Schema 可能导致解析错误 |
| 工具名称 | 工具名称区分大小写，调用时使用精确名称 |
| JSON 格式 | 参数必须是紧凑的单行 JSON，不能有格式化空格 |

## 故障排除

<Accordion title="未生成工具调用" value="no-tool-call">
**可能原因及解决方案**：

| 原因 | 解决方案 |
|------|----------|
| 注入模式为 `never` | 更改为 `auto` 或 `smart` |
| 客户端检测跳过了注入 | 禁用客户端检测或使用 `always` 模式 |
| 工具定义无效 | 验证工具定义是有效的 JSON Schema |
| 模型不支持 | 检查模型是否支持工具的复杂度 |

<Accordion title="参数不正确" value="wrong-params">
**解决方案**：

| 方法 | 说明 |
|------|------|
| 简化 Schema | 减少嵌套层级，使用基本类型 |
| 添加描述 | 为参数添加更具描述性的说明 |
| 检查 JSON | 确保 JSON 是紧凑的（无格式化） |
</Accordion>

<Accordion title="响应中出现协议文本" value="protocol-text">

| 原因 | 解决方案 |
|------|----------|

| 提示词变体不匹配 | 尝试不同的提示词变体 |
| 模型输出格式异常 | 检查模型是否正确响应提示词 |
</Accordion>

<Accordion title="客户端已注入提示词" value="client-injected">

| 方法 | 说明 |
|------|------|

| 使用 `never` 模式 | 完全禁用服务端注入 |
| 禁用客户端检测 | 强制服务端注入（可能重复） |
</Accordion>
