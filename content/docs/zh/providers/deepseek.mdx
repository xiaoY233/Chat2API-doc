## 支持的模型

| 模型 | 描述 |
|------|------|
| DeepSeek-V3.2 | 最新 DeepSeek 模型，具有增强功能 |
| DeepSeek-Search | 支持联网搜索的模型 |
| DeepSeek-R1 | 深度思考推理模型 |
| DeepSeek-R1-Search | 支持联网搜索的深度思考模型 |

## 功能特性

- 深度思考模式，支持复杂推理
- 联网搜索集成
- 流式/非流式输出
- 多轮对话支持

## 认证方式

DeepSeek 支持 **OAuth 登录认证** 和 **User Token 认证** 两种方式。

### 方式一：OAuth 登录认证（推荐）

<Steps>
  <Step title="添加供应商">
    在 Chat2API 中点击**添加供应商**，选择 DeepSeek。
  </Step>
  
  <Step title="选择认证方式">
    选择 **OAuth 登录** 方式。
  </Step>
  
  <Step title="浏览器登录">
    点击**开始登录**，系统会自动打开浏览器跳转到 DeepSeek 登录页面。
  </Step>
  
  <Step title="完成登录">
    在浏览器中完成登录后，返回 Chat2API，系统会自动获取认证信息。
  </Step>
</Steps>

### 方式二：手动获取 Token

<Steps>
  <Step title="访问 DeepSeek">
    前往 [DeepSeek Chat](https://chat.deepseek.com/) 并登录您的账户。
  </Step>
  
  <Step title="打开开发者工具">
    发起任意对话，然后按 `F12` 打开开发者工具。
  </Step>
  
  <Step title="导航到 Application">
    在开发者工具中，转到 **Application** 标签（某些浏览器为 **存储**）。
  </Step>
  
  <Step title="找到 Local Storage">
    在左侧边栏展开 **Local Storage**，点击 `https://chat.deepseek.com`，找到 `userToken` 并复制其值。
  </Step>
</Steps>

<Callout type="warning">
  请勿分享您的 Token 给他人。Token 提供了对您 DeepSeek 账户的完全访问权限。
</Callout>

## 添加到 Chat2API

1. 导航到 **供应商** > **添加供应商**
2. 从列表中选择 **DeepSeek**
3. 粘贴 `userToken` 值
4. 点击 **验证** 进行验证
5. 保存供应商

### 模型映射推荐

| 模型名称 | 实际模型 | 描述 |
|----------|----------|------|
| `DeepSeek-V3.2-Think` | DeepSeek-V3.2 | 深度思考模型 |
| `DeepSeek-V3.2-Search` | DeepSeek-V3.2 | 联网搜索模型 |
| `DeepSeek-V3.2-Think-Search` | DeepSeek-V3.2 | 深度思考 + 联网搜索 |

## 故障排除

### Token 无效

如果看到 "Token invalid" 错误：
1. 重新登录 DeepSeek 网站
2. 从 Local Storage 获取新的 Token
3. 在 Chat2API 中更新 Token
