## 供应商管理

### 添加供应商

1. 从侧边栏导航到**供应商**
2. 点击**添加供应商**按钮
3. 从列表中选择内置供应商
4. 输入认证凭证
5. 点击**验证**确认凭证有效
6. 保存供应商

### 供应商状态

每个供应商显示：
- **在线/离线**：供应商 API 是否可访问
- **活跃账户**：已配置的活跃账户数量
- **延迟**：供应商响应时间

### 账户管理

每个供应商可以有多个账户：
- 添加多个账户实现负载均衡
- 设置每个账户的每日限额
- 监控账户状态和使用情况

## 认证方式

不同供应商使用不同的认证方式：

| 认证类型 | 描述 | 存储位置 |
|----------|------|----------|
| User Token | 直接用户认证令牌 | Local Storage |
| Refresh Token | 长期刷新令牌 | Cookies |
| JWT Token | JSON Web Token | Local Storage / Cookies |
| SSO Ticket | 单点登录票据 | Cookies |

## 获取凭证

<Callout type="info">
  获取每个供应商凭证的详细说明，请参阅具体的供应商文档页面。
</Callout>

一般流程：
1. 登录供应商的网页界面
2. 打开浏览器开发者工具（F12）
3. 导航到 Application 标签
4. 在 Local Storage 或 Cookies 中找到令牌
5. 复制令牌值

## 账户状态类型

| 状态 | 描述 |
|------|------|
| 活跃 | 账户正常工作 |
| 未激活 | 账户已禁用 |
| 已过期 | 令牌已过期 |
| 错误 | 认证失败 |

## 最佳实践

1. **多账户**：为每个供应商配置多个账户实现负载均衡
2. **每日限额**：设置合理的每日限额避免触发限制
3. **定期验证**：定期验证令牌确保仍然有效
4. **安全存储**：凭证在本地 `~/.chat2api/accounts.json` 中加密存储
